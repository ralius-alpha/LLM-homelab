# ROADMAP

> LLM-homelab プロジェクトの全体ロードマップ

###############################
# [IMPORTANT] 業務利用不可     #
###############################

本ロードマップで扱う全ての構成・手順は **個人のプライベート用途専用** です。
詳細は [README.md](./README.md) および [docs/disclaimer.md](./docs/disclaimer.md) を参照してください。

## 連載全体マップ

```text
シーズン1: 基礎構築編（全12話）
  └─ 眠ってるPCでローカルAI環境を構築する
      ├─ 第1部: 基礎構築（第1-3話）
      ├─ 第2部: 外部アクセス・API（第4-6話）
      ├─ 第3部: エージェント化（第7-9話）
      └─ 第4部: 応用・カスタマイズ（第10-12話）

シーズン2: AI協働開発編（全12話・構想中）
  └─ Claude Code × ローカルLLM のハイブリッド運用
      ├─ 第1部: Claude Code 基礎
      ├─ 第2部: MCP連携
      ├─ 第3部: ハイブリッドワークフロー
      └─ 第4部: エージェント協調
```

## 対象スペック凡例

各話には対象スペックを明記します。読者は自分のPC環境に応じて適切な話を選択できます。

| 表記   | 意味                                                       |
|--------|------------------------------------------------------------|
| `A`    | クラスA（軽量環境: VRAM 6-8GB / RAM 16GB）で快適に対応     |
| `B`    | クラスB（標準環境: VRAM 12-16GB / RAM 32-64GB）で快適に対応|
| `C`    | クラスC（ハイエンド: VRAM 24GB+ / RAM 64GB+）で快適に対応  |
| `A*`   | クラスAは軽量代替手順で対応可能（本編とは別パス）          |
| `A/B/C`| 全クラスで対応可能                                         |
| `B/C`  | クラスBおよびC推奨（クラスAは動作困難または非対応）        |

各クラスの詳細スペックは [docs/architecture.md](./docs/architecture.md) を参照。

## シーズン1: 基礎構築編

### 第1部: 基礎構築（1〜3話）

#### 【第1話】序章：なぜ今、自宅にAIを置くのか

- **対象スペック**: `A/B/C`（全員向け）
- 連載全体の見取り図
- ローカルLLMとクラウドAIの違い
- 「次世代エージェント」とは何か
- 必要なPCスペック目安（クラスA/B/C分類）
- **自分のクラス判定方法**（重要）
- **今月のゴール**: 自分のPC環境を把握する
- **次号予告**: いよいよAIを召喚

#### 【第2話】最初のローカルAIを召喚する

- **対象スペック**: `A/B/C`
- Ollama / LM Studio インストール
- 最初のモデルダウンロード〜対話
- GPU/CPU動作確認方法
- **クラス別モデル選定ガイド**
  - クラスA: Llama 3.2 3B / Qwen2.5 3B
  - クラスB: Qwen2.5 7B / 14B
  - クラスC: Qwen2.5 32B / 72B
- **今月のゴール**: ターミナルで日本語対話成功
- **次号予告**: ブラウザから使えるようにする

#### 【第3話】ブラウザUIで快適に使う

- **対象スペック**: `A/B/C`
- Open WebUI 導入
- チャット履歴・モデル切替・システムプロンプト
- 複数モデル使い分け（軽量/高性能/Vision）
- **今月のゴール**: ChatGPT風UIで自分のAIと対話
- **次号予告**: 外出先から使えるようにする

---

### 第2部: 外部アクセス・API（4〜6話）

#### 【第4話】どこからでもアクセス：Cloudflare Tunnel

- **対象スペック**: `A/B/C`（スペック非依存）
- なぜポート開放しないのか（セキュリティ観点）
- Cloudflare Tunnel セットアップ
- 認証レイヤーの追加（Cloudflare Access）
- **今月のゴール**: スマホから自宅AIにアクセス
- **次号予告**: APIとして叩く

#### 【第5話】APIで叩く・自作スクリプトから使う

- **対象スペック**: `A/B/C`
- Ollama の OpenAI 互換API
- Python から呼ぶサンプル
- 簡単な自動化スクリプト例
- **今月のゴール**: 自作コードからAIを呼べる
- **次号予告**: AIに「見る目」を与える

#### 【第6話】マルチモーダル：画像を理解させる

- **対象スペック**: `B/C`
- **クラスA向け注記**: Vision対応モデルはVRAM要求が高く、
  クラスAでは実用的な動作が困難。この回はスキップ推奨、
  または最軽量Visionモデルでの参考動作のみに留める。
- Vision対応モデル導入（Qwen2.5-VL 等）
- 写真の説明・OCR・図表読み取り
- プライベート活用アイデア
- **今月のゴール**: 撮った写真をAIに解説させる
- **次号予告**: AIに「耳」を与える

---

### 第3部: エージェント化（7〜9話）

#### 【第7話】音声入力：Whisperで文字起こし

- **対象スペック**: `A*/B/C`
- **本編**: Whisper-large-v3 で高精度文字起こし（クラスB/C向け）
- **軽量代替**: Whisper-small で実用的な文字起こし（クラスA向け）
- Faster-Whisper 導入
- 音声メモ・動画の文字起こし
- LLMと組み合わせて要約自動化
- **今月のゴール**: 音声メモが自動でテキスト化＆要約
- **次号予告**: AIに「記憶」を与える

#### 【第8話】RAG入門：個人ナレッジベース構築

- **対象スペック**: `A*/B/C`
- **本編**: 高精度埋め込みモデル + 中型LLMでのRAG（クラスB/C向け）
- **軽量代替**: 軽量埋め込みモデル（all-MiniLM等）+ 小型LLM（クラスA向け）
- ベクトルDB（Chroma 等）の概念
- 自分のメモ・読書記録を食わせる
- 「自分専用に育ったAI」の第一歩
- **今月のゴール**: 自分のデータに基づいて答えるAI
- **次号予告**: AIに「手足」を与える

#### 【第9話】Function Calling：AIに道具を持たせる

- **対象スペック**: `A/B/C`
- Function Calling の仕組み
- 簡単なツール実装
- 「エージェント」の本質的な体験
- **クラス別モデル選定**:
  - クラスA: Qwen2.5 7B（Function Calling対応）
  - クラスB/C: より高精度なモデルでの安定動作
- **今月のゴール**: AIが自分で道具を選んで使う体験
- **次号予告**: ノーコードで組む

---

### 第4部: 応用・カスタマイズ（10〜12話）

#### 【第10話】Dify / n8n でノーコード構築

- **対象スペック**: `A/B/C`
- ワークフロー型AIアプリの設計
- 複数AIを連携させる
- プライベート用途のテンプレ集
- **今月のゴール**: 自分専用ワークフロー1個完成
- **次号予告**: モデルを育てる

#### 【第11話】モデルカスタマイズ入門

- **対象スペック**: `A/B/C`
- システムプロンプト設計
- Modelfile（Ollama）でキャラ作り
- 軽量ファインチューニングの概念
- **今月のゴール**: 自分好みのAIキャラを作る
- **次号予告**: 集大成

#### 【第12話】総集編：1年間で得たもの・次のステップ

- **対象スペック**: `A/B/C`
- 連載全体の振り返り
- 読者構成事例紹介（コメント募集）
- 最新ローカルAI動向
- 次の学習リソース
- **今月のゴール**: 自分の理想構成を言語化
- **次号予告**: シーズン2始動

## シーズン2: AI協働開発編（構想中）

[NOTE] シーズン1完走後に詳細化します。以下は現時点の構想。

### コンセプト

**「Claude Code × ローカルLLM」** のハイブリッド開発スタイルを実証し、
業務外プロジェクトを通じてAI協働開発の感覚を掴む。

### 想定トピック（順序未定）

- Claude Code の導入と基本ワークフロー
- ローカルLLM を MCP サーバー化
- Claude Code からローカルLLM を呼ぶ
- 役割分担設計（クラウドAI vs ローカルAI）
- 個人ナレッジベースと Claude Code の連携
- マルチエージェント協調
- 完全ローカルでのコード生成パイプライン
- セルフホスト型開発環境の構築
- プライバシー重視のAI開発フロー
- AIペアプロのベストプラクティス
- 業務外プロジェクトでの Claude Code 活用事例
- 次世代の開発スタイルとは

[NOTE] シーズン2は Claude Code (クラウドAI) を使うため、
ローカルLLM側のスペック要求はシーズン1より柔軟。
ただし MCP サーバー化する場合は一定以上のスペックが望ましい。

詳細はシーズン1進行中に深化させていきます。
